## Tech And Structure

This portfolio is intentionally lightweight and deploys as a static site.

- `index.html` - main standalone portfolio page
- `images/` - profile and homepage snapshot assets
- `.nojekyll` - ensures GitHub Pages serves the project as a plain static site

The project does not require a build step or framework runtime.

## Deployment

The site is configured to work well with **GitHub Pages**.

### Publish To GitHub Pages

1. Push the repository to GitHub.
2. Open the repository on GitHub.
3. Go to `Settings` > `Pages`.
4. Under `Build and deployment`, choose `Deploy from a branch`.
5. Set the source to the `main` branch and `/ (root)`.
6. Save and wait for deployment to finish.

### GitHub Pages URL Format

- Project site: `https://<your-github-username>.github.io/<repository-name>/`
- User site: `https://<your-github-username>.github.io/`

If the repository name is exactly `<your-github-username>.github.io`, GitHub Pages will publish it at the root user URL.
